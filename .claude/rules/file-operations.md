# ファイル操作ガイドライン

## 基本原則

ファイル操作は**慎重かつ保守的**に行う。不可逆な操作は避け、常にユーザーの意思を確認する。

## ルール

### 1. ファイルの削除禁止

- **ファイルを勝手に削除しない**
- 不要と判断したファイルがあっても、自己判断で削除しない
- 削除が必要な場合は、必ずユーザーに確認を取り、明示的な許可を得る

### 2. 不要ファイルはリネームで対応

- 不要になったファイルは削除ではなく**リネーム**で対応する
- リネーム規則: `.bak` や `.old` などのサフィックスを付与
  - 例: `config.json` → `config.json.bak`
- リネーム前にもユーザーへの確認を推奨

### 3. プロジェクト外ファイルの読み取り制限

- プロジェクトルート外のファイルを読む場合は、**事前にユーザーに確認**を取る
- 確認時にはファイルパスと読み取りの目的を明示する
- 例外: `~/.claude/` 配下のルール・設定ファイルは確認不要

### 4. プロジェクト外ファイルの変更禁止

- **プロジェクトルート外のファイルを絶対に変更しない**
- 編集・書き込み・上書きすべて禁止
- 例外: ユーザーが明示的にプロジェクト外のファイルの変更を指示した場合のみ許可

## チェックリスト

ファイル操作を行う前に以下を確認:

- [ ] 対象ファイルはプロジェクトルート内か？
- [ ] 削除操作ではなく、編集またはリネームで対応できるか？
- [ ] プロジェクト外のファイルを操作する場合、ユーザーの明示的な許可があるか？
- [ ] 操作の目的と影響範囲をユーザーに説明したか？
