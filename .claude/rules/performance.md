# パフォーマンスガイドライン

## 最適化原則

### 1. 早すぎる最適化は避ける

- まず動くコードを書く
- 計測してからボトルネックを特定
- 証拠に基づいて最適化

### 2. 一般的な最適化手法

- キャッシュを適切に活用
- データベースクエリの最適化（N+1問題の解消）
- 遅延ロード（Lazy Loading）
- コード分割（Code Splitting）
- 画像最適化（WebP、圧縮）

### 3. 計測ツール

| 対象 | ツール |
|------|--------|
| ブラウザ | Lighthouse、DevTools Performance |
| Node.js | `console.time()`, clinic.js |
| データベース | EXPLAIN ANALYZE |

## パフォーマンス改善の手順

1. **計測**: 現状のパフォーマンスを数値化
2. **特定**: ボトルネックを見つける
3. **仮説**: 改善策を立案
4. **実装**: 最小限の変更で改善
5. **検証**: 改善効果を計測で確認

## アンチパターン

- 計測せずに最適化する
- 全体を一度に最適化しようとする
- 可読性を犠牲にした過度な最適化
- 推測に基づく最適化
